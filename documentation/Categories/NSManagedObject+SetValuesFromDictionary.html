insertObjectWithDictionary:inContext:dateFormatter:

Inserts an object to the specified context and popuplates it with the provided dictionary. Does not attempt to unique the object, so ensure this is done prior to this call.

+ (instancetype)insertObjectWithDictionary:(NSDictionary *)dictionary inContext:(NSManagedObjectContext *)context dateFormatter:(NSDateFormatter *)formatter

Parameters

dictionary

The non-nil dictionary with the values to update from

context

The managed object context to insert to

formatter

Date formatter to use for date values or nil if dates are either integer values or not expected

updateValuesFromDictionary:dateFormatter:

Takes a dictionary and populates the object with values from that dictionary. Assumes that dictionary keys are the same as the object’s property names unless keyPathsForProperties returns a non-nil dictionary

- (void)updateValuesFromDictionary:(NSDictionary *)dictionary dateFormatter:(NSDateFormatter *)formatter

updateValuesFromDictionary:dateFormatter:nilUnsetValues:

As above, but allows you to specify whether keys not found in the update dictionary should be nil’d out on the managed object or not. If not, they will be left as before the update.

- (void)updateValuesFromDictionary:(NSDictionary *)dictionary dateFormatter:(NSDateFormatter *)formatter nilUnsetValues:(BOOL)nilUnset

Parameters

dictionary

The non-nil dictionary with the values to update from

formatter

Date formatter to use for date values or nil if dates are either integer values or not expected

nilUnset

Sets whether we should we nil the managed object’s values if not found in the dictionary.
